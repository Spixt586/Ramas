// Aprendiendo a trabajar con ramas

INIT

git init

Inicializa un nuevo repositorio local en la carpeta actual.

REMOTE

git clone

Descarga una copia exacta de un repositorio remoto.

STAGE

git add .

Prepara los cambios (staging) para ser incluidos en el próximo commit.


IMPORTANTE: el punto al final es para incluir todos los proyectos, si no se pone el punto al final se tendrá que poner los nombres de cada archivo uno por uno
NEW BRANCH

git commit -m

Va a permitir crear un mensaje descriptivo sobre los cambios en el código.

SWITCH

git cheackout

Nos permite movernos entre las ramas previamente creadas.

LIST

git branch

Al ejecutar este código nos mostrará las ramas creadas.

LIST

git branch ||nombre||

Crea una nueva rama.

IMPORTANTE: No confundir con git branch que nos permite visulizar una lista de las ramas creadas
RENAME

git branch -M

Renombra la ramna actual al nombre que se le indique.

SAVE

git cheackout -b

Crea una rama y nos mueve automaticamente a esa rama.

MERGE

git merge

Une el trabajo hecho en una rama a la rama que nos encontremos actualmente.
